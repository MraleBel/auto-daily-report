# Product

## Register

product

## Users
独立开发者、小团队负责人，以及需要同时跟进多个 Git 仓库进展的工程人员。他们通常在一天结束前、站会前，或准备向上同步进展时打开这个桌面工具，希望快速得到可直接复制的日报文本，而不是手动翻看多个仓库和提交记录。

## Product Purpose
Auto Daily Report 是一个面向桌面场景的日报生成工作台。它负责托管多个远程 Git 仓库、本地同步分支与作者信息，并根据选定时间范围自动生成纯文本日报。默认模式直接整理提交信息，进阶模式则接入兼容 OpenAI 的模型分析 diff，帮助用户更快产出可用、可信、可复制的日报内容。成功标准是：在一个稳定、低负担的界面里，让用户用最少步骤完成“选择仓库、检查范围、生成日报、汇总输出”这条主链路。

## Brand Personality
克制、清晰、有人味。这个产品不应像高饱和的 AI 宣发页面，也不应像冷冰冰的企业后台。它应该像一张被认真排版过的工作台，安静、可靠、略带编辑感，让用户觉得内容是被妥善整理过的，而不是被炫技包装过的。

## Anti-references
- 不要像花哨 AI SaaS 营销页：大面积炫彩渐变、夸张数据英雄区、过多悬浮卡片、为了“未来感”而牺牲任务清晰度。
- 不要像传统企业后台：僵硬的表格布局、机械分栏、满屏边框和状态色。
- 不要像开发者玩具：过度终端化、霓虹暗色、过强的 hacker 审美。

## Design Principles
- 先服务任务，再表达气质：用户应该先看懂流程，再感受到风格。
- 编辑感来自排版与节奏，不来自装饰：通过层级、留白、深浅表面切换建立品质感。
- 强调“整理后的结果”：仓库输出与汇总输出必须看起来可信、稳定、便于复制。
- 珊瑚色只用于决定性动作：主按钮和关键确认保留品牌电压，其余区域保持克制。
- 让多仓库工作变得平静：复杂度可以存在，但界面必须帮用户消化，而不是放大它。

## Accessibility & Inclusion
至少满足 WCAG AA 对比度要求。键盘导航、焦点态、禁用态、加载态都需要清晰可辨。减少不必要动画，并尊重 `prefers-reduced-motion`。长仓库名、长 URL、空状态、无报告状态都要有稳定处理，避免因为内容长度或缺失而破坏布局。
